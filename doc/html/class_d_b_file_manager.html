Detailed Description

Database file manager.

Database is implemented as a filesystem using Boost's filesystem library. Each user has a their own directory containing two files, one for user info and one for bridge info. Both files are read to create a User object. A user directory and info file are created on registration and never modified. Bridge files are created on adding a new bridge and are modified whenever a bridge is updated or added.
